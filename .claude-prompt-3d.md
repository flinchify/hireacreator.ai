## Technical requirements:
- ALL 3D components must be client-only: use "use client" directive
- ALL must be wrapped in dynamic(() => import(...), { ssr: false }) when used in pages
- ALL must have Suspense fallback (loading spinner or empty div)
- Use React.lazy or next/dynamic for code splitting
- Performance: use low-poly models, limit draw calls
- Mobile: either show simplified version or hide 3D entirely (check window.innerWidth or use a media query)
- Add a helper components/3d/canvas-wrapper.tsx that handles the common Canvas setup with:
  - Suspense boundary
  - Error boundary (falls back gracefully if WebGL unavailable)
  - Performance monitoring (limit FPS on low-end devices)
  - Responsive sizing

## Color palette for 3D:
- Primary surfaces: white (#ffffff) to light grey (#f5f5f5)
- Edges/wireframes: neutral-300 (#d4d4d4)
- Glow/accent: neutral-900 (#171717) or subtle warm (#fbbf24 amber-400) for highlights
- Environment: soft studio lighting, no harsh shadows
- Background: transparent (overlays on page background)

## Performance safeguards:
- Add a check: if the device is low-end (navigator.hardwareConcurrency < 4 or mobile), skip 3D and show static alternatives
- Use drei's useDetectGPU() or similar to detect WebGL capability
- Set Canvas frameloop="demand" where possible (only render when something changes)
- Use instanced meshes for repeated geometry
