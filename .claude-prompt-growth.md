Build three features: Quick-reply templates, Creator verification badges, and AI-powered bio writer.

## Feature 1: Quick-Reply Templates

Pre-saved response templates for creators to respond to common enquiries faster.

### DB:
```sql
CREATE TABLE IF NOT EXISTS reply_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  category TEXT, -- 'pricing', 'availability', 'intro', 'decline', 'custom'
  use_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_reply_templates_user ON reply_templates(user_id);
```

Migration at db/add-reply-templates.ts, run it.

### Seed default templates:
When a user first visits the templates section, if they have 0 templates, auto-create 4 defaults:
1. "Thanks for your enquiry" — intro/pricing template
2. "I'm currently booked" — availability template
3. "Here's my process" — intro template
4. "Not the right fit" — polite decline template

### API:
Create /api/profile/reply-templates (app/api/profile/reply-templates/route.ts):
- GET: list all templates for auth user (sorted by use_count DESC)
- POST: create new template { title, content, category }
- PATCH: update template { id, title?, content?, category? }
- DELETE: delete by id

### Dashboard UI:
Add "Reply Templates" section in dashboard:
- List of templates as cards showing title + preview of content
- Click to expand/edit
- "New Template" button
- Edit form: title, category dropdown, content textarea with variable placeholders like {name}, {service}
- Delete button with confirmation
- Copy button that copies content to clipboard

## Feature 2: Creator Verification Badge

A trust badge system. Free but requires verification steps.

### DB:
```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS verification_status TEXT DEFAULT 'unverified';
-- Values: unverified, pending, verified, rejected
ALTER TABLE users ADD COLUMN IF NOT EXISTS verification_submitted_at TIMESTAMPTZ;
ALTER TABLE users ADD COLUMN IF NOT EXISTS verification_notes TEXT;
```

Migration at db/add-verification.ts, run it.

### Verification requirements (displayed to user):
1. Email verified (already have this via auth)
2. At least 1 social account connected with 100+ followers
3. Profile photo uploaded
4. Bio written (min 50 characters)
5. At least 1 service listed

### API:
Create POST /api/profile/verify (app/api/profile/verify/route.ts):
- Auth required
- Checks all requirements above against the user's profile
- If all met, sets verification_status to 'verified' (auto-verify for now, no manual review)
- If not met, returns which requirements are missing
- Returns: { verified: boolean, missing: string[] }

### Dashboard UI:
Add a "Verification" card in the dashboard:
- If unverified: show checklist of requirements with green check / grey for incomplete
- "Get Verified" button that calls the API
- If all requirements met, immediately shows the verified badge
- If verified: show the badge with "Verified Creator" text

### Profile display:
The creator profile already has a VerifiedBadge component that shows when creator.isVerified is true. Update the assembleCreator function in lib/queries.ts to set isVerified based on verification_status === 'verified'.

## Feature 3: AI-Powered Bio Writer

One-click bio generation based on creator's existing profile data.

### API:
Create POST /api/profile/generate-bio (app/api/profile/generate-bio/route.ts):
- Auth required
- Reads the user's profile: name, category, services, socials, location
- Calls OpenAI API (or use the OPENAI_API_KEY from env, check if it exists)
- If no OpenAI key available, use a template-based approach instead:
  - Generate a professional bio from the data using string templates
  - Example: "{name} is a {category} creator based in {location}. Specializing in {service1}, {service2}, and {service3}, they bring {years?} of experience to every project."
- Return: { bio: string, alternatives: string[] } (generate 3 variations)

Actually, just use template-based generation for now (no external API dependency):
- Short bio (1-2 sentences)
- Medium bio (3-4 sentences)  
- Long bio (full paragraph)
- Each variation uses different tones: professional, friendly, creative

### Dashboard UI:
Add an "AI Bio Writer" button near the bio editing section in the dashboard:
- Button: "Generate Bio" with a sparkle/wand icon
- Clicking it calls the API
- Shows 3 generated bios in cards
- Each has a "Use This" button that fills the bio field
- User can then edit further before saving

### Design:
- The generated bios appear in a clean modal or expandable section
- Each variation has a label: "Professional", "Friendly", "Creative"
- Subtle animation when bios appear
