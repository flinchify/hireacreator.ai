Build two major dashboard features: Analytics Dashboard and Earnings Tracker.

## Feature 1: Analytics Dashboard

Add an Analytics tab/page to the creator dashboard at /dashboard/analytics (or as a tab within /dashboard).

### DB changes needed:
Create a `profile_views` table:
```sql
CREATE TABLE IF NOT EXISTS profile_views (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  creator_id UUID NOT NULL REFERENCES users(id),
  viewer_ip TEXT,
  referrer TEXT,
  user_agent TEXT,
  country TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_profile_views_creator ON profile_views(creator_id, created_at);
```

Create a `link_clicks` table:
```sql
CREATE TABLE IF NOT EXISTS link_clicks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  creator_id UUID NOT NULL REFERENCES users(id),
  link_type TEXT NOT NULL, -- 'social', 'bio_link', 'service', 'website'
  link_target TEXT NOT NULL,
  referrer TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_link_clicks_creator ON link_clicks(creator_id, created_at);
```

Create a `enquiry_log` table (to track enquiries for analytics):
```sql
CREATE TABLE IF NOT EXISTS enquiry_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  creator_id UUID NOT NULL REFERENCES users(id),
  enquirer_name TEXT,
  enquirer_email TEXT,
  budget TEXT,
  project_type TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_enquiry_log_creator ON enquiry_log(creator_id, created_at);
```

Write a migration script at db/add-analytics.ts that creates these tables. Run it against the database using the DATABASE_URL from .env.

Also update the POST /api/enquiry route to log enquiries to enquiry_log when sending.

Update the existing /api/profile/view route to log to profile_views instead of (or in addition to) just incrementing a counter.

### Analytics Dashboard UI:
Create at app/dashboard/analytics/page.tsx (or component):

Show these metrics:
- **Profile Views** — total + chart (last 30 days, grouped by day)
- **Link Clicks** — total + breakdown by link type
- **Enquiries** — total + last 30 days trend
- **Top Referrers** — list of top 10 referrer domains
- **Views by Country** — if we have country data (can skip for now)

Use simple bar/line charts. Don't use a chart library — build simple CSS-based charts:
- For the 30-day trend: a row of vertical bars (divs) with height proportional to count
- For top referrers: horizontal bar chart

### API routes:
Create GET /api/dashboard/analytics (app/api/dashboard/analytics/route.ts):
- Auth required (use existing cookie auth pattern)
- Returns: { views: { total, daily: [{date, count}] }, clicks: { total, byType: [{type, count}] }, enquiries: { total, daily: [{date, count}] }, topReferrers: [{domain, count}] }
- Daily data for last 30 days

### Navigation:
Add an "Analytics" link in the dashboard sidebar/navigation. Check existing dashboard layout for the nav pattern and add to it.

## Feature 2: Earnings Tracker

### DB: Use existing bookings table
Check the existing bookings table schema. It should have creator_id, amount, status, created_at etc.

If there's no `payout_status` column on bookings, add it:
```sql
ALTER TABLE bookings ADD COLUMN IF NOT EXISTS payout_status TEXT DEFAULT 'pending';
-- Values: pending, processing, paid, failed
ALTER TABLE bookings ADD COLUMN IF NOT EXISTS payout_date TIMESTAMPTZ;
```

### Earnings Dashboard UI:
Create at app/dashboard/earnings/page.tsx:

Show:
- **Total Earned** (all time) — big number at top
- **This Month** — current month earnings
- **Pending** — unpaid/pending bookings total
- **Available** — ready for payout
- **Earnings Chart** — last 6 months bar chart (CSS-based, no library)
- **Recent Transactions** — table/list of recent bookings with: date, client name, service, amount, status (badge: completed/pending/paid)

### API route:
Create GET /api/dashboard/earnings (app/api/dashboard/earnings/route.ts):
- Auth required
- Returns: { totalEarned, thisMonth, pending, available, monthlyData: [{month, amount}], recentBookings: [{id, clientName, service, amount, status, date}] }

### Navigation:
Add "Earnings" link in dashboard nav alongside Analytics.
