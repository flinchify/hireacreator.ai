## Feature 1: Featured Creator Rotation

Homepage spotlights that rotate weekly to incentivize creators to complete their profiles.

### DB:
```sql
CREATE TABLE IF NOT EXISTS featured_rotations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  creator_id UUID NOT NULL REFERENCES users(id),
  week_start DATE NOT NULL,
  week_end DATE NOT NULL,
  position INTEGER DEFAULT 0,
  reason TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(creator_id, week_start)
);
CREATE INDEX idx_featured_rotations_week ON featured_rotations(week_start, week_end);
```

Migration at db/add-featured-rotations.ts, run it.

### Logic:
- Each week, 4 creators are featured on the homepage
- Selection criteria (auto-rotate): prioritize creators who have completed profiles (bio, avatar, services, socials), have the verification badge, haven't been featured recently
- Create a function in lib/queries.ts: getFeaturedCreatorsRotation() that:
  1. First checks featured_rotations for current week entries
  2. If none exist for this week, auto-selects 4 creators based on: profile completeness score, verification_status, recency of last feature, and randomness
  3. Inserts them into featured_rotations and returns them

### Homepage integration:
Update the existing featured creators section on the homepage (components/homepage-content.tsx). Find where featured creators are displayed and use getFeaturedCreatorsRotation() instead. Add a small badge/label: "Featured This Week" on each card.

### Admin:
Add a "Featured Creators" section in the admin panel (if one exists at /admin or /dashboard admin tab) to manually override featured creators for the current week.

## Feature 2: Referral Program

Creators invite other creators. When the referred creator signs up and completes their profile, the referrer gets a month of Pro free.

### DB:
```sql
CREATE TABLE IF NOT EXISTS referrals (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  referrer_id UUID NOT NULL REFERENCES users(id),
  referred_id UUID REFERENCES users(id),
  referral_code TEXT UNIQUE NOT NULL,
  status TEXT DEFAULT 'pending',
  reward_granted BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);
CREATE INDEX idx_referrals_referrer ON referrals(referrer_id);
CREATE INDEX idx_referrals_code ON referrals(referral_code);
```

Add to users table:
```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS referral_code TEXT UNIQUE;
ALTER TABLE users ADD COLUMN IF NOT EXISTS referred_by UUID REFERENCES users(id);
```

Migration at db/add-referral-program.ts, run it.

### Referral code generation:
When a user first views their referral page, auto-generate a code if they don't have one. Format: first 3 chars of slug + random 4 chars, e.g., "sop-a3k9".

- POST /api/referrals/track: track when someone signs up with a referral code

### Signup integration:
Update the signup flow to accept an optional referral code (query param ?ref=CODE or input field).
Check existing auth/signup code and add referral tracking.

### Dashboard UI:

## Feature 3: Portfolio Templates

Pre-built case study layouts that creators can use to showcase their work more professionally.

### Templates (4 types):
1. "Before/After" - Side-by-side comparison layout
2. "Campaign Results" - Metrics-focused with stats and outcomes
3. "Process Breakdown" - Step-by-step workflow showcase
4. "Client Story" - Testimonial-driven narrative format

### DB:
```sql
ALTER TABLE portfolio_items ADD COLUMN IF NOT EXISTS template_type TEXT DEFAULT 'standard';
-- Values: standard, before_after, campaign_results, process_breakdown, client_story
ALTER TABLE portfolio_items ADD COLUMN IF NOT EXISTS template_data JSONB DEFAULT '{}';
-- Stores template-specific data like metrics, before/after images, steps, etc.
```

Migration at db/add-portfolio-templates.ts, run it.

### Template data structures (stored in template_data JSONB):
- before_after: { beforeImage, afterImage, description }
- campaign_results: { metrics: [{label, value, change}], summary }
- process_breakdown: { steps: [{title, description, image?}] }
- client_story: { clientName, clientCompany, quote, results }

### Dashboard editing:
In the portfolio section of the dashboard, when adding/editing a portfolio item:
- Add a "Template" dropdown: Standard, Before/After, Campaign Results, Process Breakdown, Client Story
- Based on selection, show relevant fields
- Standard: just the normal image/video upload (existing behavior)
- Before/After: two image URL inputs + description
- Campaign Results: add metrics (label + value + optional change %) + summary
- Process Breakdown: add steps (title + description per step)
- Client Story: client name, company, quote, results summary

### Profile display:
Update the portfolio section on creator profiles to render based on template_type:
- Standard: current grid display (unchanged)
- Before/After: slider or side-by-side layout with hover to swap
- Campaign Results: card with big metric numbers and a summary
- Process Breakdown: vertical timeline layout
- Client Story: quote-style card with client info

### API:
Update existing portfolio API routes to handle template_type and template_data fields.
