CRITICAL: Fix ALL mobile layout issues across the entire site. The site looks zoomed in and broken on mobile phones. This is the #1 priority — most users are on mobile.

## The Problem
On mobile phones (375-430px width), the site appears zoomed out/cramped because elements overflow the viewport width, causing the browser to shrink everything to fit.

## Root Cause Investigation
Check EVERY component and page for elements that could be wider than 100vw on mobile:
1. The header nav — desktop links may be showing on tablets/phones
2. Fixed-width elements (px values instead of percentages)
3. Flex containers without flex-wrap
4. Grid layouts with too many columns on mobile
5. Absolute positioned elements extending beyond viewport
6. Tables without overflow-x-auto
7. Long unbreakable text (URLs, emails, code blocks)

## Fixes Required:

### 1. Global CSS (app/globals.css)
Add to html and body:
```css
html, body {
  max-width: 100vw;
  overflow-x: hidden;
}
```

### 2. Layout (app/layout.tsx)
Ensure viewport meta has:
- width: device-width
- initial-scale: 1
- maximum-scale: 1 (prevents pinch zoom that causes layout issues)

### 3. Header (components/header.tsx)
The desktop nav links MUST be hidden below 1024px (lg breakpoint). Check:
- All `md:flex` should be `lg:flex`
- All `md:hidden` should be `lg:hidden`  
- Mobile hamburger toggle: `lg:hidden` (show on everything below 1024px)
- Mobile menu dropdown: `lg:hidden`
- The header container: on mobile it should be simpler — just logo left, hamburger right
- Header width: `w-[calc(100%-2rem)]` is fine but verify max-w doesn't cause issues
- On mobile, use `w-[calc(100%-1rem)]` for tighter fit

### 4. Dashboard (components/dashboard-content.tsx)
- Outer container: add `max-w-[100vw] overflow-x-hidden` 
- The desktop sidebar (aside with w-[220px]): MUST be `hidden lg:flex` not showing on any mobile/tablet
- Profile header section: MUST be `hidden lg:block`
- Mobile profile card: ensure all children use `min-w-0` and `truncate` to prevent overflow
- URL bar: text must truncate, not overflow
- Quick action buttons scroll container: ensure padding-right so last button visible
- Stats grid: `grid-cols-2` with `gap-2` on mobile (not gap-3)
- All cards: `overflow-hidden` to prevent children from breaking out
- Bottom nav: verify it doesn't add width

### 5. Homepage (components/homepage-content.tsx)  
- Hero text: `text-3xl` on mobile base, scale up with breakpoints
- All section padding: `px-4` on mobile, `sm:px-6`, `lg:px-8`
- Comparison table: must have `overflow-x-auto` wrapper
- Featured creators grid: `grid-cols-1` on mobile, `sm:grid-cols-2`, `lg:grid-cols-4`
- 3D section: `hidden lg:block` (completely hidden on mobile)
- Floating platform icons in hero: `hidden sm:block` (hidden on mobile)
- All text with `text-6xl` or `text-7xl`: must have mobile-first smaller sizes

### 6. Creator Profile (app/creators/[slug]/page.tsx)
- Cover: `h-32 sm:h-48 lg:h-72`
- Avatar: `w-16 h-16 sm:w-28 sm:h-28 lg:w-36 lg:h-36`
- Name: `text-xl sm:text-2xl lg:text-3xl`
- Grid layout: stack on mobile (`grid-cols-1 lg:grid-cols-3`)
- Portfolio: `grid-cols-1 sm:grid-cols-2`
- All badges/pills: `flex-wrap` so they wrap on narrow screens

### 7. Link-in-Bio Editor (components/link-in-bio-editor.tsx)
- Preview panel: `hidden lg:block`
- Editor: full width on mobile
- Tab bar: horizontal scroll with `overflow-x-auto`
- All inputs: `w-full` not fixed width
- Color swatches: wrap with `flex-wrap`

### 8. Browse Page  
- Filter bar: stack vertically on mobile or horizontal scroll
- Creator cards: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
- Search input: full width

### 9. Pricing Page
- Tab pills: `overflow-x-auto` wrapper
- Cards: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
- Comparison tables: `overflow-x-auto` with sticky first column

### 10. For-Brands, For-Creators pages
- Step cards: `grid-cols-1 md:grid-cols-3`
- All grids: start with 1 column on mobile
- Code blocks: `overflow-x-auto text-xs` on mobile

### 11. Leaderboard
- Table: responsive — card layout on mobile, table on desktop
- Or horizontal scroll with sticky rank column

### 12. Blog pages
- Content: `max-w-full` with `break-words`
- Images: `max-w-full`

## Testing approach:
After making changes, verify by checking:
1. No element has a computed width > viewport width
2. No horizontal scrollbar appears
3. All text is readable without zooming
4. All buttons are tappable (44px+ targets)
5. Build passes: run `npx next build`

## IMPORTANT:
- Make mobile the BASE styles (no prefix), desktop uses sm:/md:/lg: prefixes
- Every grid should start as 1 column on mobile
- Every flex row should have flex-wrap on mobile
- Every table needs overflow-x-auto
- No fixed pixel widths on mobile (use w-full, max-w-full, etc.)
