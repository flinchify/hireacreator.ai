Redesign the mobile dashboard and link-in-bio editor for a simple, easy mobile-first experience. Most users are on phones.

## PART 1: Mobile Dashboard (components/dashboard-content.tsx)

The dashboard needs to feel like a native mobile app, not a desktop app squeezed onto a phone.

### Overview section on mobile:
- Hero card: show avatar, name, and slug in a compact card at top
- Quick stats: 2x2 grid of large tap-friendly stat cards (Views, Enquiries, Earnings, Bookings) with big numbers
- Quick actions: horizontal scroll row of pill buttons (Edit Profile, Edit Socials, Add Service, View Page) instead of a grid
- Verification banner: if unverified, show a simple colored banner with "Get Verified" button
- Testimonials and Reply Templates: hide these behind "Show more" expandable sections on mobile to reduce scroll length

### Bottom navigation improvements:
The current bottom nav has 6 items which is a lot. Keep it but:
- Make icons slightly larger (20x20 instead of 16x16)
- Active state: filled icon + bold text + subtle background pill
- Add a subtle top border highlight on active item
- Labels: increase to text-[10px] from text-[9px]

### Mobile editing sheets:
All edit flows (profile, socials, services) should use full-screen bottom sheets on mobile:
- Slide up from bottom
- Top has a drag handle bar (thin grey line) 
- Title + close X button
- Content scrolls independently
- Save button: sticky at bottom of sheet, full width, always visible
- Min touch targets: 48px for all buttons and inputs

### Services section on mobile:
- Each service as a simple card with: title, price, and "Edit" button
- "Add Service" as a prominent full-width button at top
- Package editing: collapsible accordion within each service card

### Calendar section on mobile:
- Simple toggle for calendar enabled/disabled
- Session types as a clean list with swipe-to-edit or tap-to-expand
- Bookings below as a simple timeline/list

### Analytics on mobile:
- Big stat numbers at top in a 2x2 grid
- Chart: simplified, maybe just a sparkline or mini bar chart that fits narrow width
- Top referrers: simple numbered list

### Earn section on mobile:
- Referral link with big "Copy" button
- Stats as large number cards
- Boost button prominent

## PART 2: Link-in-Bio Editor (components/link-in-bio-editor.tsx)

This editor needs to be dead simple on mobile. Think of how Instagram's profile editor works.

### Mobile layout:
Currently it's a side-by-side editor+preview on desktop. On mobile:
- NO side preview panel — use a floating "Preview" button instead
- Preview opens as a full-screen overlay when tapped
- Editor takes full width on mobile
- Sections as horizontal scrollable tabs at top (not a sidebar)

### Tab navigation on mobile:
Replace the sidebar section nav with a horizontal scroll tab bar:
```
[Links] [Style] [Background] [Buttons] [Animation] [Templates]
```
- Horizontal scroll, no wrapping
- Active tab: underline + bold
- Sticky at top below the header

### Simplify each section:

#### Links tab:
- Social links: each as a simple row with platform icon + handle + delete X
- "Add Social" button
- Bio links: drag handle + title + URL in compact rows
- Simple, list-based, easy to reorder

#### Style tab:
- Template picker: horizontal scroll of template previews (small cards you tap to select)
- Font picker: dropdown, not a grid
- Accent color: simple color swatches row (6-8 preset colors + custom input)
- Text color: same

#### Background tab:
- Three options as big tap cards: "Solid Color" / "Gradient" / "Image"
- Selected option expands to show its controls
- Gradient: preset gradient swatches to tap
- Image: big "Upload" button area

#### Buttons tab:
- Shape: visual row of 4 shape options (rounded, pill, square, outline) as tappable cards
- Animation: row of animation previews

#### Animation tab:
- Intro animation: list of animations with preview on tap
- Owned vs locked clearly shown

#### Templates tab:
- Grid of template previews (2 columns)
- Tap to select, immediate apply

### Preview button:
- Floating action button (FAB) in bottom-right: round, 56px, with eye icon
- Tapping opens full-screen preview overlay
- Preview has a phone frame mockup
- Close button (X) at top-right of preview

### Save behavior:
- Auto-save on every change (debounced 500ms)
- Show tiny "Saved" toast at top when auto-saved
- No manual save button needed (reduces cognitive load)

### General mobile rules:
- All inputs: full width, min-h-[48px], large text (text-base not text-sm)
- Color pickers: large touch-friendly swatches (min 44x44px)
- Upload buttons: large dropzone areas
- Spacing: generous padding between sections (py-6)
- Animations: smooth transitions between tabs (slide left/right)
