Ensure users who create paid services must have Stripe payments connected. If they try to add a service with a price > 0, prompt them to connect Stripe first.

## 1. Check Stripe connection status

In the dashboard, when a user tries to add or edit a service with price > 0:
- Check if user has stripe_account_id set
- If not, show a prompt/modal: "Connect Stripe to accept payments" with a button that starts the Stripe Connect onboarding
- Don't block free services (price = 0 / "Open to offers") — those don't need Stripe

### In the services editing flow (components/dashboard-content.tsx or ServicesSheet):
Before saving a service with price > 0, check:
```ts
if (price > 0 && !user.stripe_account_id) {
  // Show Stripe connect prompt instead of saving
}
```

### Stripe Connect button:
The button should call POST /api/stripe/connect which creates a Stripe Connect onboarding link. Check if this endpoint already exists. If not, create it:
- Create a Stripe Connect Express account for the user
- Save the stripe_account_id to the users table
- Return the onboarding URL
- After onboarding, Stripe redirects back to the dashboard

Check existing Stripe code in app/api/stripe/ or app/api/checkout/ to see the pattern.

## 2. Dashboard indicator

In the dashboard overview, if user has no Stripe connected, show a banner:
"Connect Stripe to accept payments for your services"
With a "Connect Stripe" button.

If already connected, show a small green checkmark: "Payments connected via Stripe"

## 3. Profile display

On creator profiles (app/creators/[slug]/page.tsx), if a creator has services with price > 0 but no Stripe connected:
- The "Book" button should show "Contact Creator" instead (since payment can't be processed)
- OR: disable the booking flow and show "Coming soon" on the price
