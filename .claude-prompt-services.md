Build two features: Service Packages (tiered pricing) and Testimonial Imports.

## Feature 1: Service Packages (Basic/Standard/Premium tiers)

Currently each service is a single flat price. Add tiered packages like Fiverr does.

### DB changes:
Create a `service_packages` table:
```sql
CREATE TABLE IF NOT EXISTS service_packages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  service_id UUID NOT NULL REFERENCES services(id) ON DELETE CASCADE,
  tier TEXT NOT NULL CHECK (tier IN ('basic', 'standard', 'premium')),
  title TEXT NOT NULL,
  price INTEGER NOT NULL DEFAULT 0,
  delivery_days INTEGER NOT NULL DEFAULT 7,
  revisions INTEGER DEFAULT 1,
  features JSONB DEFAULT '[]',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(service_id, tier)
);
CREATE INDEX idx_service_packages_service ON service_packages(service_id);
```

Write a migration at db/add-service-packages.ts and run it.

### API changes:
1. Update GET /api/profile/services to also return packages for each service
2. Create POST /api/profile/services/packages (or nest under existing service routes):
   - Add/update packages for a service
   - Body: { serviceId, packages: [{ tier, title, price, deliveryDays, revisions, features }] }
   - Auth required, must own the service
3. Update the service display on creator profiles

### Creator profile display (app/creators/[slug]/page.tsx sidebar):
When a service has packages, show a tabbed interface within each service card:
- Three tabs: Basic | Standard | Premium
- Each tab shows: title, price, delivery time, revisions, feature list
- Book button shows the selected package price
- If no packages exist, show the service as-is (backward compatible)

### Dashboard editing:
In the dashboard service editing flow, add package management:
- When editing a service, show "Add Packages" toggle
- When enabled, show 3 package forms (Basic, Standard, Premium)
- Each has: title, price, delivery days, revisions, features (comma-separated input)
- At minimum require the Basic tier if packages are enabled

### Checkout:
Update the booking checkout to include the package tier. Update /api/checkout/booking to accept an optional `packageId` param. If provided, use that package's price instead of the service base price.

## Feature 2: Testimonial Imports

Let creators add testimonials to their profile — either typed or from screenshots.

### DB:
```sql
CREATE TABLE IF NOT EXISTS testimonials (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  client_name TEXT NOT NULL,
  client_company TEXT,
  client_avatar TEXT,
  content TEXT NOT NULL,
  rating INTEGER CHECK (rating BETWEEN 1 AND 5),
  source TEXT, -- 'manual', 'google', 'trustpilot', 'screenshot'
  screenshot_url TEXT,
  display_order INTEGER DEFAULT 0,
  is_visible BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_testimonials_user ON testimonials(user_id);
```

Migration at db/add-testimonials.ts, run it.

### API:
Create /api/profile/testimonials (app/api/profile/testimonials/route.ts):
- GET: return all testimonials for the authenticated user
- POST: create a new testimonial (auth required)
  - Body: { clientName, clientCompany?, content, rating?, source, screenshotUrl? }
- DELETE: delete a testimonial by id
- PATCH: update order or visibility

### Dashboard UI:
Add a "Testimonials" section in the dashboard:
- List existing testimonials with edit/delete
- "Add Testimonial" button opens a form:
  - Client name (required)
  - Company (optional)
  - Rating (1-5 stars, clickable)
  - Testimonial text (required)
  - Source dropdown: "Written by me", "Google Review", "Trustpilot", "Other"
  - Screenshot upload (optional) — for now just accept a URL input since blob storage isn't connected yet

### Creator profile display:
Add a "Testimonials" section on the creator profile page (app/creators/[slug]/page.tsx):
- Show after the About section and before Portfolio
- Display as elegant quote cards with client name, company, rating stars
- Only show if the creator has testimonials
- Query: create getCreatorTestimonials(userId) in lib/queries.ts
